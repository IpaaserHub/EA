Phase 2: 本番環境（SaaS基盤）構築依頼書

現在ローカル（localhost）で稼働しているAIサーバーを、一般ユーザーへ配布可能な「Web API構成」へ移行するための要件です。

1. システム構成の変更

Current (現状): Localhost構成

User PC: [ MT5 EA ] <--> [ Python Server (Local) ]

Target (目標): Client-Server構成

User PC A: [ MT5 EA ] --(Internet)--> \

User PC B: [ MT5 EA ] --(Internet)-->  [ Cloud Python Server (VPS/AWS) ]

User PC C: [ MT5 EA ] --(Internet)--> /

2. 開発・構築タスク

A. サーバーデプロイ (Deployment)

Pythonサーバー（main_advanced.py）を常時稼働のクラウドサーバー（VPS推奨）上に設置してください。

推奨環境: Vultr (Tokyo), AWS Lightsail, または Xserver VPSなど

必須設定:

HTTPS化: MT5のWebRequestはHTTPS推奨のため、SSL証明書（Let's Encrypt等）の設定。

プロセス管理: サーバーが落ちても自動復帰するように systemd や Docker で管理。

B. 認証機能の強化 (Authentication)

現在は ALLOWED_ACCOUNTS リストをコード内にベタ書きしていますが、これを外部データベース（SQLiteまたはPostgreSQL）で管理できるようにしてください。

管理機能: 口座IDの「追加」「削除」「有効期限設定」ができる簡易スクリプト、またはAPIを用意してください。

C. EA側の接続先変更

EAの ServerURL の初期値を http://127.0.0.1:8000 から、構築したクラウドサーバーのURL（例: https://api.ai-ea-system.com）に変更してコンパイルしてください。

3. ゴール

私の手元のPCにあるEA（URL変更済み）から、ザマンさんが構築したクラウド上のPythonサーバーに接続し、シグナルを受信できること。

私のPCのPythonを停止していても、EAが稼働すること。
